PROJECTS

MIT PROJECT

The MIT project, originally designed by Scott Young in 2010?? was an experiment to see if you could learn everything in MIT's Computer Science B.S. program in under a year (4x faster!). I was inspired to make my own curriculum and learning process to see if I could improve on the idea, and from where I stand, I have succeeded.

"The biggest waste of time is to do well something that we need not do at all" -Gretchen Rubin

I set my sights on becoming a software developer, not a computer scientist and therefore realized: many of the classes offered at MIT were useful, but only in the hands of someone who already knows how to write code.

If you have a project which demands complex mathematical models, then 3 courses in advanced math is justified, but if not, you're putting the cart before the horse. All recreational learning is time well spent, but fun learning in a profitable domain is time BETTER spent.

My process consists of 4 steps.
1. Planning
2. Accumulation
3. Sortation/reduction
4. Action


WIKIWORLD

The goal of WikiWorld is to make academic research into a game that anyone can play, to make the world of knowledge fun and accessible, and to spread enthusiasm for learning.

WIKIWORLD consists of 4 parts:
1. Data Accumulation: "Node wikipedia scraper"
2. Organization: MongoDB JSON files, tagging and categorization
3. Display: (Obsidian for now) C++ quadtrees native window and mobile app
4. Navigation: GUI "exploration game" loop with each page you learn giving you skills and items
This is my longest term project, I will have more information as I complete omre of the project!
